# Сравнение RabbitMQ и Redis
## Цель: Оценка пропускной способности и стабильности брокеров при разных размерах сообщений.

## Ключевые выводы

### Производительность: 
* Redis значительно обходит RabbitMQ на высоких нагрузках. При целевых 5 000 RPS Redis выдал почти максимум, в то время как RabbitMQ просел в 10 раз.

### Большие сообщения (100 KB): 
* RabbitMQ не справился: зафиксировано более 80% ошибок. Вероятно, сработал flow control или переполнение памяти.

* Redis стабилен: задержки выросли, но потерь данных нет.

### Latency:
* На малых объемах задержки идентичны. При росте нагрузки Redis сохраняет предсказуемость, тогда как RabbitMQ начинает либо сильно тормозить, либо "отваливаться".

## Итог

Redis — лучший выбор для High-throughput систем и простых очередей (скорость и стабильность под нагрузкой).

RabbitMQ — только для сложных сценариев маршрутизации, но требует жесткого контроля лимитов и ресурсов, так как "из коробки" быстро деградирует на тяжелых данных.
